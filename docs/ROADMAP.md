## 待开发

### 中优先级

- [ ] **配额感知**: 基于已知日配额上限（10,000/天）做预算提示；当远程工具返回 `quota_exhausted` 时停止并提示等待，不再承诺“每次操作前实时检查剩余额度”。
- [ ] **竞品达人搜索**: 输入竞品品牌名，复用 `search_creators` 与现有评分模型筛选高潜力名单；hashtag 搜索仅作为实验性补充，不作为默认能力承诺。

### 低优先级

- [ ] **相似达人推荐**: 基于 `search_creators` + `creator_performance` 实现“以某达人为模板找相似达人”的 MVP 能力；方案细节见 `docs/tiktok-creator-similar-finder-research.md`。

## 暂不做

- **消息监听（达人回复接收/分类/状态机）**: TikTok 无 GET 轮询，接收依赖 Webhook (Type 33)，且需消息落库；本包无数据库、禁止数据修改，暂不纳入本包范围。
- **A/B 测试建联话术**: 追踪回复率需要持久化发送记录和回复结果，超出当前包“无自有数据库、禁止数据修改”的约束。
- **导出功能**: 当前包聚焦对话式筛选、分析和建联流程，不再单独承诺 CSV/Excel 导出能力。

## 跨包依赖

- **`create_conversation` MCP 工具**（批量建联前置）已沿依赖链补齐：sdk (`createConversation`) → mcp-remote (`/mcp/tools/create-conversation`) → mcp-server (`create_conversation` 映射)。
- **评分模型字段** 已由底层链路补齐；本包按固定展示契约消费 `creator_performance` 字段。
- **多店铺切换** 的底层授权与店铺选择模型已由 `mcp-remote` / `mcp-server` 提供；本包负责将自然语言意图映射到对应的 `authorize` 调用。
