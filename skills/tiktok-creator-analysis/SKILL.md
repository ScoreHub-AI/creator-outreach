# TikTok Creator Analysis — 达人表现分析与评分

获取达人近 30 天表现数据，按 100 分制评分模型综合评估并排序。

## 触发条件

当用户提到以下意图时加载本技能：
- "分析这些达人的带货表现/数据"
- "帮我评估达人的带货能力"
- "达人的粉丝画像怎么样"
- "给这些达人打分/排名"

## 执行方式（MCP 优先）

对候选达人**逐个**调用 MCP 工具 **`creator_performance`**（入参 `creator_user_id`），拿到表现数据后应用
[评分模型](./references/scoring-model.md) 打分排序。批量分析属多步操作，先与用户确认名单再执行。

如果 ScoreHub MCP 工具不可见、工具列表为空、连接已关闭或本地 MCP 未启动，直接提示用户：“ScoreHub 服务暂时未连接，请完全退出并重新打开 Claude Code 或 WorkBuddy 后重试；如仍无法使用，请联系 ScoreHub 支持。”不要向终端用户提及 Node.js、npm、网络、环境变量、TikTok API 凭证或直连 API，也不要调用 `authorize`。

如果工具返回结构化 JSON，优先按 `error_type` 做分流，不要靠自然语言错误文本猜测。

如果本地登录状态正常，但工具提示当前店铺的 TikTok 授权失效或异常，直接提醒用户去 ScoreHub 重新绑定该店铺后再试；不要展示错误码，也不要要求用户重复执行 `authorize`。

如果工具提示请求过于频繁、持续限流或配额受限，直接提醒用户耐心等待、减少本次分析人数或稍后重试；不要建议“重新授权拿一个干净的 token”，也不要主动走 `authorize`，除非用户自己明确要求重新授权。

### 输出（固定格式）

禁止直接展示 MCP 原始 JSON。评分是候选集内归一化后的**相对分**，按以下两种情况输出：

#### 单个达人

- 不输出综合分或五维相对分，避免单元素归一化造成虚高分数。
- 固定展示：达人身份、近 30 天 GMV/销量、直播和视频表现、粉丝数与画像、品类分布、标签、数据缺口、优势/风险、合作建议。
- 明确说明：“当前仅有 1 位达人，无法生成有意义的相对评分；至少需要 2 位候选人才可评分排名。”

#### 两个及以上达人

1. **评分摘要**：成功分析数、失败数、评分对象范围和相对评分说明。
2. **排行榜**：固定展示“排名 / 达人 / 综合分 / 带货能力 / 内容影响力 / 粉丝规模 / 品类匹配 / 粉丝质量 / 标签”。
3. **达人详情**：逐人展示关键原始指标、优势/风险、数据缺口、合作建议，以及从搜索上下文保留的标识。
4. **失败名单**：部分分析失败时单列达人和用户可理解的原因，不得把失败值作为 0 分参与排序。
5. **下一步**：建议选择达人建联、补充候选人或调整筛选，不自动发送消息。

排序与字段规则：

- 默认按综合分降序；并列时依次按带货能力分、内容影响力分、粉丝数降序，仍相同则保持原候选顺序。
- 分数均按评分模型取整并以 `/100` 展示；品类数据未授权或维度无法计算时显示 `—` 和原因，不用无说明的 0 分掩盖缺口。
- 必须保留搜索结果中的 `creator_user_id` 和 `creator_open_id` 映射。`creator_performance` 不返回 `creator_open_id`，只能从当前对话中的搜索结果透传，不能猜测或推断。
- 无法关联 `creator_open_id` 时显示“未保留建联标识”，并说明该达人暂不能直接建联；Markdown 中弱化为详情末项，HTML 中放入可展开复制区域。
- 建议必须由实际指标、维度分、相对排名和数据缺口支撑，不得编造内容风格、合作历史或转化表现。

### WorkBuddy HTML 可视化

- 成功结果为 1–5 位时使用 Markdown；达到 **6 位**时，若当前宿主提供 HTML/可视化产物能力，默认生成自包含的响应式 HTML 报告。用户明确要求 HTML 时不受人数限制。
- 宿主不支持 HTML，或 HTML 生成失败时，必须回退为包含完整排行榜、详情和失败名单的 Markdown。
- HTML 评分报告必须包含：评分摘要、可排序排行榜、五维分数、达人详情、数据缺口、合作建议和弱化的双 ID 复制区域。
- 只使用内联 CSS/JavaScript，不依赖 CDN、远程字体或外部图片；只实现真实可用的搜索、排序、展开和复制交互，不添加无法触发 Agent 或 MCP 操作的伪按钮。
- 昵称、用户名、标签及所有工具返回文本写入 HTML 前必须转义；禁止把工具内容作为 HTML 或脚本执行。
- 生成 HTML 后仍需在对话中给出成功/失败数量、前三名和数据完整性的简短摘要。只有实际生成产物时才可声明存在 HTML，不得虚构文件名或 `outputs` 路径。

## 评分模型（100 分制）

| 维度 | 权重 | 依据 |
|------|------|------|
| 带货能力 | 30% | GMV + 销量 |
| 内容影响力 | 25% | 播放/观看量 + 互动率 |
| 粉丝规模 | 15% | 粉丝数 |
| 品类匹配 | 20% | 目标品类 GMV 占比/类目重叠 |
| 粉丝质量 | 10% | 目标性别占比 + 核心年龄段占比 |

完整归一化公式、标签体系（💎带货王者 / 🎬内容高手 / 🎯精准美妆 / 📢大V 等）、以及 MCP 字段与
原始接口字段的映射说明，见 [references/scoring-model.md](./references/scoring-model.md)。

## 错误处理

| 问题类型 | 处理 |
|----------|------|
| MCP 工具不可见、工具列表为空、连接关闭或本地 MCP 未启动 | 提示“ScoreHub 服务暂时未连接，请完全退出并重新打开客户端后重试；如仍无法使用，请联系 ScoreHub 支持。”不提及技术配置，不走 `authorize` |
| `oauth_invalid` | 仅在真正未授权或本地 OAuth 失效时，才允许提示用户调用 `authorize` |
| 当前店铺的 TikTok 授权失效或异常 | 用中文提醒用户：本地登录状态正常，但当前店铺需要先到 ScoreHub 重新绑定后再试；不要展示错误码，不要要求重复 `authorize` |
| 达人当前不可分析（如无电商权限或区域不匹配） | 跳过该达人，并说明当前达人不满足分析条件 |
| 日配额已用完 | 停止 |
| 请求过于频繁或持续限流 | 先提醒用户耐心等待，建议减少本次分析人数或稍后重试；必要时有限退避重试。不要建议重新授权；只有用户明确要求时才走 `authorize` |
| 参数无效 | 提示用户修正待分析名单或输入格式，不走授权路径 |
