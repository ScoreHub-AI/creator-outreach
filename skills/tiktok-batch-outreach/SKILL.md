# TikTok Focused Creator Outreach Validation — 小范围达人建联验证

先创建会话再发送建联消息，对经过画像评估的精选候选人进行快速验证。建联不以海量群发为目标。

本文件是小范围达人建联验证专项行为的权威来源；Agent 只保留能力摘要与跨能力编排。若确认、发送、速率、输出或失败处理存在冲突，以本文件为准。

## 产品定位与规模边界

- 建联只用于验证精选候选人，不承诺海量群发；当用户提出大规模发送要求时，引导其先搜索、分析、排名并确定精选小名单。
- 完成画像评估前不主动建议大量建联。

## 触发条件

当用户提到以下意图时加载本技能：
- "给这些达人发消息/建联/联系"
- "批量发送建联消息"
- "创建会话并发消息给达人"
- "向达人发送合作邀请"

## 执行前必须确认

多位达人建联验证属多步操作，发送前**必须**与用户确认：
1. **品牌名**（模板变量 `{brand}`）
2. **精选达人名单**
3. **整批单选的 Offer 类型及其实际金额 / 比例**
4. **通知栏前 15 字预览与所有将轮换的完整最终话术**

### WorkBuddy Offer 选择

- 用户提出建联但未给出完整 Offer 时，调用 `AskUserQuestion`，设置 `multiSelect: false`，提供“现金 + 高佣 / 高佣 + 免费送样 / 独家折扣码 + 免费送样”三个选项；每个选项的描述直接展示默认前缀和前 15 字预览。默认值、变量和前缀只从 [建联话术模板](./references/message-templates.md) 读取，不在本 Skill 重复。
- 允许用户通过自定义输入修改预设金额或比例。只有在修改值可唯一识别且用户确认后才继续；不得猜测币种、金额、佣金或折扣。
- 用户已给出可执行 Offer 时不二次询问。同一批达人只能选择一种 Offer，不按达人或模板轮换不同承诺。
- 取消、空选择或自定义值存在歧义时，不调用 `create_conversation` 或 `send_message`。`AskUserQuestion` 不可用或失败时，降级为三个带编号的文本选项。
- 用户明确提供完整最终文案时不静默改写。若原文未将利益置于前 15 字，先展示合规的利益前置改写，等待用户明确选择原文或改写版。系统生成或改写的文案必须满足 Reference 的前 15 字规则。

## 执行方式（MCP 优先，两步建联）

对每个达人：

1. **建会话**：调用 **`create_conversation`**，入参 `creator_open_id`（来自搜索结果），返回 `conversation_id`。
2. **发消息**：调用 **`send_message`**，入参 `conversation_id` + `msg_type: "TEXT"` + `content`（模板填充后的文本）。

授权、限流、MCP 不可用和本地运行环境恢复遵循 Agent 的权威规则，本技能不重复展开。

### 速率与话术

- 每条发送间隔 **5-8 秒 + 随机抖动**（间隔 <5s 会触发 429）。
- Offer 预设、通知栏前 15 字规则、话术正文、轮换规则和变量定义的唯一权威来源是 [建联话术模板](./references/message-templates.md)，本 Skill 不重复模板细节。
- 建联是用户确认后的即时操作，不支持定时或跨时区调度。用户要求未来时间发送时，说明边界并请其在合适时间重新发起。

### 输出

实时进度（每条 ✅/❌ + conversation_id + message_id）+ 最终汇总（成功/失败/跳过）。

## 关键陷阱

- `create_conversation` 的入参名是 **`creator_open_id`**，其值与表现分析兼容入参 `creator_user_id` 使用的是同一个搜索结果 ID。
- `creator_open_id` **仅来自搜索接口**，分析接口不返回——务必从搜索结果透传，不要构造第二套达人 ID。

## 错误处理

建联专属错误分支仅处理 `creator_open_id` 无效、`create_conversation` 参数不合法、消息内容不合法，以及单达人消息配额或发送条件不满足；其余授权、限流、MCP 可用性与本地环境问题均遵循 Agent 权威规则。
